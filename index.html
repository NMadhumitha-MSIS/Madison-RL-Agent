What this project does

The source selection problem

🌍

The problem

Madison agents query multiple real-world sources (Wikipedia, arXiv, Reddit, DuckDuckGo) for different topics. Naive agents waste queries on irrelevant sources. RL learns which source to trust per context.

📈

UCB Contextual Bandit

Approach 1. Learns a Q-value table per context-source pair. Balances exploration of untried sources with exploitation of known good ones using the UCB1 formula. Fast convergence, no GPU needed.

🎯

REINFORCE Policy Gradient

Approach 2. Learns a full probability distribution over sources per context. Uses Monte Carlo returns with a variance-reducing baseline. Policy entropy tracks how decisive the agent becomes over time.

🆕

Real API grounding

Unlike simulated environments, this system fetches live data from 5 real free APIs. Authentic reward noise from network failures and empty results makes the RL task genuinely challenging.

Mathematical formulation

Two RL approaches

Approach 1 -- UCB Contextual Bandit

UCB(s,a,t) = Q(s,a) + c * sqrt( ln(t_s) / N(s,a) ) Q(s,a) += (r - Q(s,a)) / N(s,a)

Q(s,a) = mean reward for source a in context s. N(s,a) = visit count. c = 1.5 exploration constant. Guarantees sublinear regret O(sqrt(K * T * ln T)).

Approach 2 -- REINFORCE Policy Gradient

pi(a|s) = softmax(theta_s) theta += alpha * (G_t - b(s)) * grad log pi(a|s)

G_t = discounted return (gamma = 0.95). b(s) = running mean baseline reduces variance. alpha = 0.05 learning rate. Entropy H(pi) tracks decisiveness.

Reward function

R(s,a) = r_success + r_length + r_relevance clipped to [-1.0, 2.0]

r_success = +1.0 / -1.0. r_length = +0.5 if words > 50. r_relevance = 0.3 * keyword overlap ratio. Designed to penalize silent failures.

MDP formulation

S = {AI, ML, Climate, Genomics, Quantum, Economics, Health} |S| = 7 A = {Wiki, arXiv, Reddit x2, DDG} |A| = 5

Non-stationary environment: context shifts each episode. Transitions driven by user query topic. Reward is immediate (no lookahead needed for bandits).
